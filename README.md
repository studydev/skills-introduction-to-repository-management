# 저장소 관리 소개

_협업 친화적이고 건강한 프로젝트를 지원할 수 있는 GitHub 기능의 기본을 배워보세요._

## 환영합니다

- **대상**: 협업을 시작해야 하는 개발자
- **배울 내용**: 더 많은 사람이 협업자로 참여할 때 저장소 콘텐츠를 보호하는 다양한 방법
- **만들 것**: 추가 교사들이 안전하게 협업할 수 있도록 Mergington 고등학교의 방과후 활동 웹사이트 저장소를 준비합니다.
- **사전 요구사항**:
  - Skills 실습: [GitHub 소개](https://github.com/skills/introduction-to-github)
  - Skills 실습: [마크다운으로 소통하기](https://github.com/skills/communicate-using-markdown)
  - Skills 실습: [Pull Request 리뷰하기](https://github.com/skills/review-pull-requests)
- **소요 시간**: 이 실습은 1시간 이내에 완료할 수 있습니다.

이 실습에서 다음을 수행합니다:

1. 간단한 규칙 세트와 설정을 추가하여 저장소 콘텐츠를 제한합니다.
2. 협업자를 안내하는 절차를 전달합니다.
3. 코드의 특정 부분에 대한 책임을 특정 협업자에게 할당합니다.
4. 개인 저장소와 조직 저장소에서의 협업 차이를 배웁니다.
5. 건강한 협업 환경을 촉진하기 위한 기본 규칙을 수립합니다.
6. 보안 업데이트를 관리하는 프로세스를 수립합니다.

> [!IMPORTANT]
> 이 실습은 여러 GitHub 기능의 개요를 제공하기 위한 것입니다.
> 각 주제에 대해 더 자세히 배울 수 있는 참고 자료를 제공하지만, 특정 주제에 대한 상세한 설명은 포함하지 않습니다.
